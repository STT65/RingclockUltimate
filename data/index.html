In Manual Brightness, the automatic light sensor is disabled, giving you direct, fixed control over the master brightness of the entire clock.

  • Value 255: The display runs at full, unthrottled brightness (only restricted by your global Power Limit).
  • Value 1 - 254: The overall light output is proportionally dimmed down.
  • Value 0: The display is completely turned off.

In automatic mode, the clock dynamically adjusts its LED brightness based on the room's ambient light intensity. Here is how the parameters work together:

  • Ambient Light [Lux]: Displays the current ambient light intensity measured by the sensor in real-time.
  • Max Lux Threshold: The room light level at which the display should reach its maximum brightness. If the room gets brighter than this threshold, the clock will not increase its brightness any further.
  • Max Brightness: The absolute maximum brightness level the clock will use in automatic mode.
  • Min Brightness: The minimum brightness level the clock will drop to when the room is completely dark (0 Lux).

Ambient & Markers

The Ambient layer provides static background lighting to help orient the clock face. You can enable and colorize three different marker types, which are stacked on top of each other:

  • Ambient Light: Illuminates all LEDs to create a solid, continuous background color.
  • Hour Markers: Places accent lights at every hour position. If enabled, these overwrite the base ambient color at their specific positions.
  • Quarter Markers: Places stronger accent lights at the 12, 3, 6, and 9 o'clock positions. These have the highest priority and overwrite both the hour markers and the base ambient light.

Special effects are visual highlights that overlay the standard time display. All effects run for a fixed duration and can be triggered manually or at a configurable interval.

  • Interval: The trigger logic is strictly aligned with midnight (0:00).
    Example: 1h triggers exactly at the top of every hour. 24h starts the effect exactly at midnight.
  • Intensity: Controls the strength of a effect (1 = minimum, 255 = maximum).

Stepper Motor - Mechanical Time Display

The stepper motor powers the mechanical heart of the ring clock. Adjust these settings to perfectly tune the movement dynamics to your personal preference.

  • Motor Mode: Determines what to do with the motor: off, show hours, minutes or seconds or do the homing procedure.
  • Time grid: This setting defines how the motor advances the time: Analog (smooth): The motor runs continuously at a constant speed, automatically synchronizing with the time source in the background. Grid: The motor snaps to discrete time positions based on your chosen distance. Each jump uses MaxSpeed and Acceleration settings to simulate a clean, mechanical stepping effect.
  • MaxSpeed & Acceleration: These parameters are used only for the Grid mode and define the maximum motor speed and acceleration to ensure smooth and optimal dynamics for the chosen grid mode.

Homing & Zero Calibration

–
Sensor based homing procedure:
  1. Wait until automatic homing completes (status: Ready).
  2. The hand should now be roughly at 12 o'clock.
  3. Use the jog buttons to fine-tune the position.
  4. If good, press Accept & Save to store the fine-tuning permanently.
  5. Now you can switch Motor Mode to the desired time display mode.
Manual homing procedure (without position sensor):
  1. Disconnect the clock from the power supply.
  2. Bring the motor shaft to the 12 o'clock position.
  3. Reconnect the clock to the power supply.
  4. While booting, the motor will be powered and snaps to a certain magnet position.
  5. Goto Motor Mode=Homing and use the jog buttons to fine-tune the position.
  6. If good, press Accept & Save to store the fine-tuning permanently.
  7. Now you can switch Motor Mode to the desired time display mode.

MQTT Client

Offline

Topic structure (base = Topic Base setting, default ringclock):

  • <base>/set/<key> — write a setting (e.g. ringclock/set/motorSpeed → 200)
  • <base>/get/<key> — read current value (publish to topic to request, device responds on same topic)
  • <base>/cmd/save — save settings to flash
  • <base>/cmd/reboot — reboot device
  • <base>/cmd/resync — resynchronise motor
  • <base>/status/online — 1 when connected, 0 as Last Will (retained)
  • <base>/status/lux, current_mA, rssi, uptime — published every second

Settings are saved to mqtt.json separately from settings.json so they survive a LittleFS firmware update.
